# 법무 문서 (`/legal`)

린디오퍼레이션이 운영하는 서비스들의 이용약관·개인정보 처리방침을 한 곳에서 관리한다.
제품이 늘어나면 이 폴더에 파일을 추가하고 `registry.js`에 등록하기만 하면 된다.

## URL 규칙

| 용도 | 경로 |
|---|---|
| 허브 | `/legal` |
| 현행본 (한국어) | `/legal/{제품}/{종류}` |
| 현행본 (영어) | `/legal/{제품}/{종류}/en` |
| 보관본 (한국어) | `/legal/{제품}/{종류}/v/{시행일}` |
| 보관본 (영어) | `/legal/{제품}/{종류}/en/v/{시행일}` |

`{종류}`는 `terms`(이용약관) 또는 `privacy`(개인정보 처리방침).
스토어·앱에는 **현행본 경로**를 등록한다. 개정해도 URL이 그대로 유지된다.

예) 차트드로우: `https://lindyoperation.com/legal/chart-draw/privacy`

## 동작 방식

- 각 문서는 `{ version, effective, summary, body }`의 배열이다. **시행일 오름차순**을 유지한다.
- **현행본 = 시행일(KST 00:00)이 지난 것 중 최신.** 아무것도 시행 전이면 현행본은 `null`이고,
  화면에는 "아직 시행 전" 배너와 함께 예정 본문을 보여준다.
- 시행일이 미래인 버전을 배열에 추가해두면 페이지 상단에 **시행 예정 배너**가 자동으로 뜨고
  (고지 의무 이행), 시행일 00:00에 **본문이 자동 전환**된다. 재배포가 필요 없다.
- 모든 버전은 보관본 URL에 영구 보존되고, 페이지 하단 개정 이력 표에서 열람할 수 있다.

## 새 제품 추가

1. `{제품}.terms.ko.js`, `{제품}.terms.en.js`, `{제품}.privacy.ko.js`, `{제품}.privacy.en.js` 작성
   (`chart-draw.*` 파일을 복사해 쓰면 된다)
2. `registry.js`의 `PRODUCTS` 배열에 항목 추가

라우트와 프리렌더 경로는 `allLegalRoutes()`가 자동 생성하므로 따로 손댈 곳이 없다.

## 개정 절차

1. **기존 버전의 `body`는 절대 수정하지 않는다.** 보관본이 훼손된다. 새 `const V1_1 = ...`을 만든다.
2. 시행일을 정한다 — **게시일 + 고지 기간**
   - 일반 변경: 7일 이상
   - 이용자에게 불리하거나 중대한 변경(권리 축소·유료화·수집 항목 확대 등):
     **30일 이상 + 회원 개별 통지(이메일)**
3. 배열 끝에 `{ version, effective, summary, body }`를 추가하고 배포한다.
4. 배포 후 시행 예정 배너와 개정 이력이 뜨는지 확인한다.
5. (중대 변경인 경우) 안내 메일을 발송한다.
6. 시행일 이후 본문이 자동 전환됐는지 확인한다.

## 주의

- **개인정보 처리방침은 코드 실체와 반드시 일치해야 한다.** 수집하지 않는 항목을 적으면
  Google Play 데이터 보안 신고·Apple 개인정보 라벨과 어긋나 오히려 심사 리스크가 된다.
  기능을 추가하기 **전에** 방침을 먼저 개정하고 시행일을 지나게 한 뒤 기능을 켜는 순서가 맞다.
- 영문판은 편의 제공용이며 각 문서에 **한국어본이 우선한다**는 문구를 넣는다.
- 마크다운은 `src/components/Markdown.jsx`가 처리하는 부분집합만 쓴다
  (`##`, `###`, 문단, `**굵게**`, `-` 목록, `|` 표, `>` 인용, `---`).
